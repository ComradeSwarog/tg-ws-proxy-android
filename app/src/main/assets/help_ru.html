TG WS Proxy — Справка

TG WS Proxy — это MTProto WebSocket мост для Android с обходом DPI (DoH, параллельный коннект, CF fallback).

Быстрый старт

  1. Настройте Хост, Порт, Секрет и соответствие DC:IP (значения по умолчанию подходят для большинства пользователей).
  2. В карточке Настройки обхода включите нужные функции (описаны ниже).
  3. Нажмите Сохранить настройки — все переключатели и значения сохраняются.
  4. Нажмите Запустить для запуска прокси.
  5. Скопируйте ссылку подключения или нажмите Открыть в Telegram, чтобы вставить её в настройки прокси Telegram.

Что нового в v1.5.0

  • Автоматически сгенерированный секрет — при первом запуске создаётся новый 32-hex секрет. Можно перегенерировать через иконку обновления рядом с полем секрета.
  • Стабильность Samsung/Android 16 — обновлённый foreground service + WakeLock refresh предотвращают «заморозку» сети системой через ~30 минут.
  • Локализация — русский и английский языки. Приложение автоопределяет язык системы; переключить вручную можно в карточке «Настройки обхода».
  • Встроенная справка — эта страница, открывается по кнопке ? в верхней панели.
  • Проверка обновлений — кнопка с иконкой обновления в панели; автопроверку при запуске можно включить/выключить в «Настройках обхода».
  • WS Frame Padding / Ротация DoH — переключатели обфускации DPI в «Настройках обхода».

Настройки

  • Хост / Порт — локальный адрес прослушивания (по умолчанию 127.0.0.1:1443).
  • Секрет — 32 hex-символа, генерируется автоматически при первом запуске. Нажмите иконку обновления рядом с полем, чтобы сгенерировать заново.
  • DC:IP — соответствие номеров дата-центров Telegram их IP-адресам (по одному в строке).
  • Fake TLS домен — домен для SNI-обфускации (obfs2), например cloudflare.com.
  • Размер пула — количество предварительно разогретых WebSocket-соединений.

Настройки обхода

  • DoH резолвинг — резолв IP Telegram через DNS-over-HTTPS вместо обычного DNS.
  • Авто Fake TLS — автоматически включает Fake TLS при блокировке прямого IP.
  • Параллельный коннект — подключается к нескольким IP одновременно для быстрого fallback.
  • Медиа через CF — направляет загрузку медиа через Cloudflare, экономя прямой трафик.
  • CF Proxy fallback — использует Cloudflare Worker прокси при отказе прямых соединений.
  • CF Proxy приоритет — сначала пробует CF перед прямым подключением.
  • WS frame padding — добавляет самоописывающий случайный padding к WebSocket-фреймам (по умолчанию выключено).
  • Ротация DoH провайдеров — циклически переключается между DoH Cloudflare, Google и Quad9.
  • Работать в фоне — оставляет прокси работающим при закрытии UI приложения.
  • Язык — Авто (системный), Русский или Английский. Переключение применится после перезапуска.
  • Автоматически проверять обновления — проверяет GitHub Releases при запуске. Отключите здесь, если предпочитаете только ручную проверку.

Логи

Откройте вкладку Логи для просмотра логов прокси в реальном времени. Можно обновить, очистить, поделиться или сохранить в папку Загрузки. Уровень логирования (Debug / Info / Warn) управляется чипами над окном логов.

Совет: Если что-то не работает, поставьте уровень логов на Debug, воспроизведите проблему и поделитесь логами.
