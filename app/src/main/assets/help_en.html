TG WS Proxy — Help

TG WS Proxy is an MTProto WebSocket bridge for Android with DPI bypass (DoH, parallel connect, CF fallback).

Quick Start

  1. Configure your Host, Port, Secret and DC:IP mappings (default values work for most users).
  2. In the Bypass Settings card enable the features you need (see below).
  3. Tap Save Settings — all toggles and values are persisted.
  4. Tap Start to run the proxy.
  5. Copy the connection link or tap Open in Telegram to paste it into Telegram proxy settings.

What's New in v1.6.1

  • Auto Network Recovery — the proxy now detects WiFi/mobile network switches and instantly resets stale connection state (cooldowns, pool, domain blacklist, WifiLock). No more manual restart after switching networks.
  • Fixed keepalive PONG timeout — empty PONG frames from Telegram servers no longer cause false keepalive timeouts that disconnected active bridges.
  • Reduced socket timeout — soTimeout lowered from 70s to 35s for faster dead-connection detection after network loss.

What's New in v1.6.0

  • Parallel Race: Direct WS vs CF Fallback — on first connect, both direct WebSocket and Cloudflare fallback are launched simultaneously. The faster one wins and bridges immediately, reducing first-connect time to 1–3 seconds.
  • CF-Proven Fast Path — after 2+ successful CF connections to a DC, direct attempts are skipped for 5 minutes. Counters decay automatically so direct paths are retried when network conditions improve.
  • Fixed thread pool deadlock — TLS handshakes now run on a dedicated thread pool, separate from the parallel connect pool. Previously they shared one pool, causing all threads to block with no thread available to execute TLS — connections could never complete.
  • Fixed slow first connect (8–20s → 1–3s) — aggressive timeout reductions: CF proxy 6s→3s, DoH 8s→3s, race cap 6.5s→4.5s, TLS handshake 8s→5s. Removed artificial 150ms CF delay. Known-blocked DCs skip race entirely and go straight to CF fallback.
  • Fixed proxy hanging after hours — removed permanent CF-fallback lock-in that blocked direct WebSocket reconnections. CF success is now decayed every 5 minutes so direct paths are retried after transient blocks.
  • Improved WebSocket pool health checks — replaced unreliable blocking PING check with simple isClosed check, preventing dead sockets from being handed back to clients.
  • WebSocket read timeout — set a 70-second read timeout on underlying TLS socket so threads are freed instead of leaking on silent network drops.
  • Service restart robustness — restart alarm now has a reliable fallback for devices that restrict exact alarms, preventing stale proxy state after a crash or Doze.

Settings

  • Host / Port — local listen address (default 127.0.0.1:1443).
  • Secret — 32 hex characters, auto-generated on first launch. Tap the refresh icon next to the field to regenerate.
  • DC:IP — mapping of Telegram DC numbers to direct IPs (one per line).
  • Fake TLS domain — domain for SNI obfuscation (obfs2), e.g. cloudflare.com.
  • Pool size — number of pre-warmed WebSocket connections.

Bypass Settings

  • DoH resolving — resolves Telegram IPs via DNS-over-HTTPS instead of plain DNS.
  • Auto Fake TLS — automatically enables Fake TLS if direct IP connections are blocked.
  • Parallel connect — connects to multiple resolved IPs simultaneously for faster fallback.
  • Media via CF — routes media downloads through Cloudflare to save direct bandwidth.
  • CF Proxy fallback — uses Cloudflare Worker proxy when direct connections fail.
  • CF Proxy priority — tries Cloudflare before direct connections.
  • WS frame padding — adds self-describing random padding to WebSocket frames (disabled by default).
  • Rotate DoH providers — cyclically switches between Cloudflare, Google and Quad9 DoH endpoints.
  • Work in background — keeps the proxy running when the app UI is closed.
  • Language — Auto (system default), Russian or English. Switch applies after restart.
  • Auto check for updates — checks GitHub Releases on startup. Disable here if you prefer manual checks only.

Logs

Open the Logs tab to view real-time proxy logs. You can refresh, clear, share or save them to the Downloads folder. Log level (Debug / Info / Warn) is controlled by the chips above the log window.

Tip: If something doesn't work, switch log level to Debug, reproduce the issue and share the logs.
